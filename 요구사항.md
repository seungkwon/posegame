# Pose Memory Game 요구사항

## 1. 제품 목표

- 브라우저에서 동작 시퀀스를 기억하고 따라 하는 포즈 메모리 게임을 제공한다.
- 포즈 추론, 동작 판정, 시퀀스 비교는 가능한 한 브라우저에서 처리한다.
- 백엔드는 사용자 로그인과 게임 결과 저장 및 조회를 담당한다.

## 2. 핵심 사용자 흐름

1. 사용자는 `user_id`만 입력해 로그인한다.
2. 게임은 목표 동작 시퀀스를 프리뷰로 보여준다.
3. 사용자는 웹캠 앞에서 같은 순서로 동작을 수행한다.
4. 시스템은 동작 시퀀스 일치 여부를 판정한다.
5. 결과는 백엔드에 저장되고 사용자는 기록 화면에서 조회할 수 있다.

## 3. 프론트엔드 요구사항

- React + Vite 기반 브라우저 앱이어야 한다.
- 로컬 개발 환경에서 `http://localhost:5173`으로 실행 가능해야 한다.
- 로그인 화면, 게임 화면, 기록 화면을 제공해야 한다.
- 웹캠 연결 UI와 카메라 권한 안내를 제공해야 한다.
- MediaPipe PoseLandmarker를 사용해 실시간 포즈 추론을 수행해야 한다.
- 포즈 랜드마크 오버레이와 FPS 표시를 제공해야 한다.
- 목표 시퀀스 프리뷰, 카운트다운, 진행률 표시를 제공해야 한다.
- 신뢰도, 입력 잠금, 기본 자세 복귀 상태를 사용자에게 보여줘야 한다.
- 카메라를 사용하지 않는 수동 테스트 버튼을 유지할 수 있어야 한다.

## 4. 동작 판정 요구사항

- 최소 지원 동작
  - 기본 자세
  - 양팔 들기
  - 왼팔 들기
  - 오른팔 들기
  - 스쿼트
  - 좌측 기울기
  - 우측 기울기
  - 점프
- 기본 자세를 기준으로 점프와 스쿼트 판정을 보정해야 한다.
- 같은 동작이 연속 중복 입력되지 않도록 잠금 또는 쿨다운이 있어야 한다.
- 다음 동작 입력 전 기본 자세 복귀 확인이 가능해야 한다.

## 5. 백엔드 요구사항

- FastAPI 기반 API를 제공해야 한다.
- PostgreSQL에 사용자와 게임 결과를 저장해야 한다.
- Docker Compose로 데이터베이스를 쉽게 실행할 수 있어야 한다.
- 지원 API
  - `GET /health`
  - `POST /auth/login`
  - `POST /games/results`
  - `GET /games/me/results`

## 6. 데이터 요구사항

### users

- `id`
- `user_id`
- `created_at`

### game_results

- `id`
- `user_id`
- `level`
- `target_sequence`
- `player_sequence`
- `is_success`
- `score`
- `played_at`

## 7. 품질 요구사항

- 프론트엔드는 `npm run build`가 성공해야 한다.
- 프론트엔드 smoke test가 가능해야 한다.
- 백엔드는 health, login, result save, history fetch 흐름을 smoke test로 검증할 수 있어야 한다.
- 루트에서 `verify.ps1`로 공통 검증을 한 번에 실행할 수 있어야 한다.

## 8. 문서 요구사항

- 빠른 실행 경로는 `README.md`에 유지한다.
- 환경 준비는 `BOOTSTRAP.md`에 유지한다.
- 단계별 체크리스트는 `SETUP_CHECKLIST.md`에 유지한다.
- 백엔드와 프런트 수동 확인 흐름은 각각 quickstart 문서에 유지한다.
