# Pose Memory Game 계획서

## 1. 개요

이 프로젝트는 브라우저 기반 포즈 메모리 게임이다. 사용자는 화면에 표시된 동작 시퀀스를 기억한 뒤 웹캠 앞에서 같은 순서로 수행한다. 프론트엔드는 포즈 추론과 동작 판정을 처리하고, 백엔드는 로그인과 결과 저장 및 기록 조회를 담당한다.

## 2. 기술 방향

### 프론트엔드

- React
- Vite
- MediaPipe PoseLandmarker

### 백엔드

- FastAPI
- SQLAlchemy
- PostgreSQL
- Docker Compose

## 3. 현재 구현 범위

- ID 기반 로그인
- 단계별 목표 시퀀스 생성
- 목표 시퀀스 프리뷰와 카운트다운
- 실시간 포즈 추론과 규칙 기반 동작 분류
- 실시간 FPS 및 신뢰도 표시
- 점프/스쿼트 보정과 기본 자세 기준선 보정
- 중복 입력 방지와 neutral reset gating
- 결과 저장 API 및 기록 조회 화면
- 프론트엔드/백엔드 smoke test
- 루트 검증 스크립트

## 4. 아키텍처 요약

### 프론트엔드 역할

- 로그인 입력 처리
- 카메라 연결과 MediaPipe 추론
- 동작 분류 결과 표시
- 목표 시퀀스와 사용자 입력 시퀀스 비교
- 결과 저장 API 호출
- 기록 화면 표시

### 백엔드 역할

- 사용자 생성 또는 로그인
- 게임 결과 저장
- 사용자 기록 조회
- health endpoint 제공

## 5. 단계별 구현 기록

### Step 1. 프로젝트 골격과 기본 흐름

- 프론트엔드/백엔드 구조 생성
- Docker Compose 추가
- 로그인, 결과 저장, 기록 조회 기본 흐름 연결

### Step 2. 포즈 추론과 동작 분류

- MediaPipe 실시간 추론 연결
- 기본 동작 분류기 추가
- 캔버스 오버레이 및 FPS 표시 추가

### Step 3. 게임 플레이 흐름 개선

- 프리뷰 단계 추가
- 카운트다운 추가
- 진행률 UI 추가

### Step 4. 판정 안정화

- 신뢰도 기준 추가
- 중복 입력 쿨다운 추가
- 상태 메시지 보강

### Step 5 이후

- UI 문구 정리
- 오픈 리소스 문서화
- 점프/스쿼트 안정화
- threshold 튜닝 구조화
- neutral reset gating
- smoke test 추가
- 검증 스크립트 추가
- 온보딩 문서와 quickstart 정리

## 6. 운영 기준

- 프론트엔드 로컬 실행: `frontend`에서 `npm run dev`
- 백엔드 로컬 실행: `backend`에서 `uvicorn app.main:app --reload`
- 전체 검증: 루트에서 `./verify.ps1`
